# 金利時系列に対するGBDT特徴量エンジニアリング知見

## プロジェクト概要

BOJ OISスワップレート（M1〜M8：第1〜第8会合先）を対象に、MPM（金融政策決定会合）前後のレート変動を予測するLightGBMモデルを構築した。バタフライ変換（B{n} = 2M{n} - M{n-1} - M{n+1}）を予測対象とした**RV Butterflyモデル**が最高精度（CS IC 3日=0.296, 5日=0.355）を達成している。

以下の知見は短期スポット金利など他の金利系列にも転用できる。

---

## 1. 目的変数の設計：絶対水準ではなく「相対価値」を予測する

### 問題

金利の絶対水準変化（`ΔM{n}`）を直接予測すると、金利カーブ全体の並行シフト（全限月が同方向に動く）が支配的となり、個別銘柄の相対的な動きを捉えられない。

- **Global IC**（予測と実績の全サンプルSpearman相関）は高く見えても
- **CS IC**（クロスセクション：同日に複数銘柄をランク付けする精度）は低くなる

CS ICは相対価値戦略（ロング/ショートペアトレード）の実用精度を正確に反映するため、**主評価指標はCS ICを使うこと**。

### 解決策：変換によって共通因子を除去する

| 変換 | 式 | 意味 |
|------|----|------|
| **スプレッド** | `M{n}_spread = M{n} - policy_rate` | 政策金利との差（水準の共通トレンド除去） |
| **隣接差分（Curve）** | `C{n} = M{n}_spread - M{n+1}_spread` | 隣接会合間のカーブ傾き |
| **バタフライ** | `B{n} = 2M{n}_spread - M{n-1}_spread - M{n+1}_spread` | 曲率（リッチ/チープ） |

バタフライ変換は並行シフトで打ち消し合い、曲率の変化だけを抽出する。**特徴量空間と目的変数空間を揃えること**（バタフライを予測するなら特徴量もバタフライベースにする）が重要。

### 目的変数の正規化

```python
Target_{h}d_norm = (B{n}_{t+h} - B{n}_t) / per_instrument_std
```

銘柄ごとの全期間stdで正規化することで異分散を補正し、プーリング時にスケール差を吸収する。逆正規化用にstdを保持しておくこと。

---

## 2. 特徴量の定常化：分数階差（Fractional Differencing）

### 問題

金利水準は非定常時系列（単純差分でICが高く、水準でICが低い）。単純な1階差分（`d=1`）にすると記憶（long-range dependence）を失う。

### 解決策：分数階差（`d=0.4`）

```python
frac_diff(series, d=0.4, window=50)
```

- `d=0` ：元の水準（非定常だが記憶あり）
- `d=1` ：1階差分（定常だが記憶なし）
- `d=0.4`：部分的に定常化しつつ記憶を保持（López de Pradoの手法）

このプロジェクトでは水準特徴量（`Fly_Level`, `M1_spread`, `Slope_M1M8`）と分数階差（`B{n}_frac_diff`等）を**両方同時に入れる**のが有効だった。

- **水準特徴量**：mean reversionの起点（現在の歪みの大きさ）
- **分数階差**：直近の変化方向・モメンタム

---

## 3. プーリング：複数系列を縦積みして1モデルで学習する

複数の金利系列（例：B2〜B7の6系列）を縦に結合し、`Meeting_Index`（銘柄ID、カテゴリ変数）を特徴量として追加して1つのGBDTモデルで学習する。

**メリット**：
- 限月間の共通パターンを学習できる
- 各系列の訓練データ量が6倍になる
- 銘柄間の相関を特徴量から暗黙的に利用できる

**注意点**：
- `Meeting_Index` はLightGBMに`categorical_feature`として明示する
- 正規化は系列ごとに行い、スケール差を吸収する

---

## 4. イベントカレンダーの活用：Days_to_MPM

```python
Days_to_MPM = 次回イベント（会合等）までの営業日数
```

これをそのまま線形特徴量として入れる。sin/cosに変換すると5日IC が -0.024 と悪化した（線形で十分）。

短期スポット金利では対応するイベントとして以下が考えられる：
- 次回FOMC/BOJ/ECB開催までの日数
- 月末・四半期末
- 重要指標発表日（CPI等）

---

## 5. コンテキスト特徴量：水準と外部指標

予測対象がカーブの曲率であっても、以下の水準情報はコンテキストとして有効だった：

| 特徴量 | 役割 |
|--------|------|
| `M1_spread` | フロントエンドの金利水準（市場全体のアンカー） |
| `Slope_M1M8` | カーブのスロープ（バタフライとスロープは相関） |
| `Actual_Policy_Rate` | 政策金利（制度的な下限） |
| `USDJPY_frac_diff` | リスクオフ/オン |
| `JGB先物_frac_diff` | 長期金利方向 |

外部指標は**分数階差**に変換して入れる。水準のままだと非定常で悪影響。

## 6. データ品質：欠損補完と補完フラグ

**MICE補完**（Multiple Imputation by Chained Equations）で欠損値を補完し、補完した箇所に`{col}_is_imputed`フラグを立ててモデルに伝える。

```python
# 補完フラグをそのまま特徴量に追加する
df['M1_is_imputed']  # 0 or 1
```

これにより、モデルが補完箇所を「不確実なデータ」として認識し、適切に重み付けできる。

---

## 7. バリデーション：Walk-Forward（Purging付き）

金融時系列では**未来情報のリーク**と**オーバーラップによるデータリーク**が精度を水増しする。

```
訓練窓（拡張） | purge=5日 | テスト窓=90日
```

- **拡張窓**：訓練データは最古から現在まで全て使う（安定性向上）
- **Purge（5日）**：訓練・テスト間の間隔。5日ホライズンで予測する場合、5日の重複を防ぐ
- **early stopping なし**：金融時系列の非定常性でval lossが即発散するフォールドが多発する。固定ラウンド数（100）の方が安定

---

## 8. 不採用となった手法と理由

| 手法 | 結果 | 理由 |
|------|------|------|
| Days_to_MPM を sin/cos 変換 | 5d IC -0.024 | 線形で十分。周期性は弱い |
| カーブ形状14特徴量を追加 | IC 大幅低下 | オーバーフィット。カーブ情報は水準特徴量に含まれている |
| Spot Spread（S{n} = M{n} - T12）を目的変数に | CS IC ≈ 0.04（ほぼゼロ） | T12が全系列共通成分のため、クロスセクションでは全系列が同方向に動きランクが無意味になる |
| バタフライ目的変数変換（別空間に変換） | IC 大幅低下 | 並行シフトで B2-B7 の値がほぼゼロになり信号消失 |
| 全ペアプーリング（28系列） vs 隣接プーリング（7系列） | IC低下 + Gap拡大 | 高相関系列を多数追加するとノイズを反復学習。シンプルな隣接差分の方が安定 |

## 短期スポット金利への適用時の注意点

このプロジェクトはOISスワップ（会合先限月）が対象だったが、**スポット金利（テナーもの）**への転用では以下を検討する：

1. **共通因子の除去方法**：隣接差分 `T{n} - T{n+1}` またはバタフライ変換が同様に有効なはず
2. **イベントカレンダー**：次回CB会合までの日数は同様に使える
3. **アンカー**：政策金利との差（スプレッド）ではなく、OIS金利全体の水準をアンカーとして使う
4. **プーリングの単位**：テナー（1M, 3M, 6M等）をプーリング単位とし、`Tenor_Index` をカテゴリ変数として追加
5. **CS ICの定義**：「同日に複数テナーをランク付けする精度」がCS IC。これが相対価値戦略の実用指標
